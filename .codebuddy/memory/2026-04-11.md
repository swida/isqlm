# 2026-04-11

## Added: Warning count display in query results

**Feature**: MySQL `mysql_warning_count()` now shown in query result summaries, matching mysql CLI output (e.g. `1 row in set, 2 warnings`).

**Changes**:
1. **C layer (`mysql-el.c`)**: Added `Fmysql_warning_count` function exposing `mysql_warning_count(conn)` as `mysql-warning-count` to Elisp.
2. **Elisp layer (`isqlm.el`)**: Added `isqlm--warning-count-string` helper returning `", N warnings"` or `""`. Applied to all 6 output points:
   - Sync SELECT: "N rows in set, W warnings"
   - Sync Empty set: "Empty set (0 rows), W warnings"
   - Sync DML: "Query OK, N rows affected, W warnings"
   - Async SELECT: same
   - Async Empty set: same
   - Async DML: same (two places: SELECT-no-result fallback + non-SELECT)

## Refactored: mysql-el error signaling — structured data like Emacs SQLite

**Motivation**: Previously `signal_mysql_error` formatted errors as a string `"ERROR 1690 (22003): ..."` and signaled generic `error`. This made it impossible for Elisp to programmatically extract errno/sqlstate, and `mysql_errno()` could be reset by the time Elisp tried to re-query it (causing "Unknown error").

**New design (modeled after Emacs `sqlite.c`)**:

### C layer (`mysql-el.c`):
1. **Defined `mysql-error` error symbol** (inherits from `error`) via `put` in `emacs_module_init`:
   - `(put 'mysql-error 'error-conditions '(mysql-error error))`
   - `(put 'mysql-error 'error-message "MySQL error")`
2. **`signal_mysql_error(env, conn)`**: Signals `mysql-error` with structured data `(ERRNO SQLSTATE ERRMSG)` — a list of (integer, string, string). No more string formatting in C.
3. **`signal_mysql_stmt_error(env, stmt)`**: Same for prepared statement errors using `mysql_stmt_errno`/`mysql_stmt_sqlstate`/`mysql_stmt_error`.
4. **Removed**: `Fmysql_errno`, `Fmysql_error`, `Fmysql_sqlstate` functions and their registrations (no longer needed).
5. **Removed**: `prefix` parameter from `signal_mysql_error`.
6. All `signal_error(env, mysql_stmt_error(stmt))` calls → `signal_mysql_stmt_error(env, stmt)`.

### Elisp layer (`isqlm.el`):
1. **New accessors**: `isqlm--mysql-error-p`, `isqlm--mysql-error-errno`, `isqlm--mysql-error-sqlstate`, `isqlm--mysql-error-errmsg` — extract from `(mysql-error ERRNO SQLSTATE ERRMSG)`.
2. **`isqlm--error-message`**: For `mysql-error`, formats as `"ERROR 1690 (22003): ..."`. For generic `error`, extracts string as before.
3. **`isqlm--output-mysql-error`**: Simplified — just calls `(isqlm--error-message err)` + newline. No longer queries the connection handle.
4. **Removed**: `isqlm--format-mysql-error` (was querying `mysql-errno`/`mysql-error`/`mysql-sqlstate` from connection).

### Tests:
- Extended `isqlm-test-error-message` with `mysql-error` signal format, predicate, and accessor tests.
